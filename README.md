# Bot-Finder
Telegram Bot for Searching and Summarizing Messages
## Overview
Bot Finder is a Telegram bot that operates via a personal Telegram account (Telethon) to read any channels where the user is a participant. It provides search by keywords for a specific date, daily summaries generated by AI, and supports multiple LLM providers (OpenAI, Gemini, Anthropic, DeepSeek, OpenRouter, Ollama, custom OpenAI‑compatible APIs). The bot uses inline keyboards for channel selection, date picking, and provider switching.

## Features
- 🔍 Search messages by keywords for a selected date and channel
- 📊 Generate AI‑based summaries of daily channel activity
- 📅 Inline calendar for date selection
- ⚙️ Provider selection with persistent storage in SQLite
- 🔒 Access control via `ALLOWED_USER_IDS`
- 🛡️ Local encryption of `.env` using Fernet + PBKDF2

## Installation
1. Clone the repository
2. Create `.env` from `.env.example` and fill required fields
3. Install dependencies: `pip install -r requirements.txt`
4. Run the bot: `python -m bot.main`

## Usage
- Use `/start` to open the main menu
- Select **Поиск сообщений** to search by keywords
- Select **Сводка за день** to generate a summary
- Open **Настройки** to switch AI providers

## Security
- Critical secrets (`API_ID`, `API_HASH`, `PHONE_NUMBER`) are stored in a protected file (`/etc/tg_bot_finder/secrets.env`) with permissions `640`
- Regular configuration (e.g., `BOT_TOKEN`) resides in `.env` which is encrypted locally using `scripts/encrypt_env.py`
- The `.env` file and session files are listed in `.gitignore` to prevent accidental exposure

## Deployment
A deployment script is provided in `deploy/setup_server.sh`. It automates:
- System user creation
- Secrets file setup
- Systemd service configuration
- Directory permissions
- Service enablement and startup
